# Zenn記事フォーマットガイド

## 記事の基本構成

Zennの技術記事は以下の構成を推奨：

1. **はじめに** - 記事の概要と執筆理由・背景
2. **対象読者** - 想定読者を明示（初心者向け/中級者向けなど）
3. **この記事を読むメリット** - 読者が得られる知見や成果
4. **結論** - 記事で最も伝えたいポイント（先に結論を提示）
5. **本文** - 詳細な内容（大項目は## h2、中項目は### h3を使用）
6. **まとめ** - 全体の振り返りと読後感を良くする締め

## Markdown記法

### 見出し
- **本文の大項目**: `## 見出し`（h2から始める - アクセシビリティ推奨）
- **本文の中項目**: `### 見出し`（h3まで）
- ネストは深くしすぎない（h4以降は避ける）

### コードブロック
```言語名:ファイル名
コード内容
```

### リスト
- 順不同: `-` または `*`
- 順序付き: `1.`, `2.`, ...

### 画像
```
![代替テキスト](画像URL =250x)
```
- 幅指定: `=○○x`でpx単位指定可能
- キャプション: 画像直下に `*キャプション*`

### Zenn独自記法
```
:::message
重要なメッセージ
:::

:::message alert
警告メッセージ
:::

:::details タイトル
折りたたみコンテンツ
:::
```

### その他
- **太字**: `**テキスト**`
- *イタリック*: `*テキスト*`
- ~~打ち消し線~~: `~~テキスト~~`
- インラインコード: `` `code` ``
- 区切り線: `-----`

## 注意点

1. **見出しは## から開始** - アクセシビリティのため
2. **適度な文字数** - 技術記事は2000文字以上推奨
3. **具体例を含める** - コードサンプルや実例を提示
4. **読みやすさ重視** - 段落を適切に分け、リストや表を活用
5. **目次の自動生成** - Zennは見出しから自動で目次を生成
