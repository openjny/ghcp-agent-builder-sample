# Zenn記事作成エージェント

Zenn向け技術記事の企画・執筆・レビューを支援するエージェントワークフロー。

## エージェント一覧

| エージェント | 絵文字 | 役割 | 成果物 |
|------------|--------|------|--------|
| zenn.planner | 📋 | トピック選定と構成案作成 | `articles/draft/{slug}/plan.md` |
| zenn.writer | ✍️ | 記事本文の執筆 | `articles/draft/{slug}/article.md` |
| zenn.reviewer | 🔍 | 品質チェックと改善提案 | `articles/draft/{slug}/review.md` |

## ワークフロー

```
1. 📋 企画立案 → 2. ✍️ 執筆 → 3. 🔍 レビュー → (修正) → 公開
   ↑______________|              |
                                 ↓
                         必要に応じて企画見直し
```

### 1. 📋 企画立案 (zenn.planner)
- トピック選定（ユーザー指定 or 自動提案）
- 対象読者とメリットの明確化
- 構成案の作成
- **受け入れ基準**: タイトル・対象読者・構成が明確、Zenn推奨フォーマット準拠

### 2. ✍️ 執筆 (zenn.writer)
- 企画書に基づいた記事執筆
- Zenn Markdown記法の正しい使用
- コードサンプル・具体例の含有
- **受け入れ基準**: 2000文字以上、企画書との整合性、見出し構成が論理的

### 3. 🔍 レビュー (zenn.reviewer)
- 企画書との整合性確認
- Markdown記法の正確性チェック
- 読みやすさ・内容の質の評価
- **受け入れ基準**: 具体的な改善提案、優先度明示、次アクション明確

## ディレクトリ構造

```
articles/
  draft/
    {slug}/
      plan.md      # 企画書
      article.md   # 記事本文
      review.md    # レビュー結果
docs/
  zenn-format.md         # Zennフォーマットガイド
  writing-guidelines.md  # 執筆ガイドライン
```

## 使い方

1. `#zenn.planner` でトピック提案または企画書作成
2. 企画承認後、ハンドオフで `#zenn.writer` へ
3. 執筆完了後、ハンドオフで `#zenn.reviewer` へ
4. レビュー結果に応じて修正または公開

## 重要ポイント

- **見出しは## から開始** - アクセシビリティ推奨
- **コードサンプル最低2つ** - 技術記事の実用性
- **2000文字以上** - 十分な情報量
- **具体例必須** - 抽象的説明のみ避ける
- **レビュープロセス必須** - 品質保証
